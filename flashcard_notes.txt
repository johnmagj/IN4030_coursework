- FASTA:
	- Name of both a file format AND a search heuristic (slower than BLAST, but better at finding distant related sequences)
	- file format extension: .fasta and .faa, the first is for both nucleotides or amino acids, the latter strictly for amino acids
- BLAST:
	- Heuristics search algo: Count only consecutive matches (cause prob. more important), reduces number of changes compared to the same number of matches but scattered. 
	- Different algorithms:
		-> seed + extend: index k-mers in query sequence -> search for similar k-mers in DB, if few matches prob. no very good match, if many prob. better -> then check adjacent sequence in between matching k-mers

	- k-mers: sequence of consecutive matches, length k
	- Case: Query sequence, we don't know if will fined the entire length as a whole
	- Protein domains: highly conserved region, rest of the protein could be very different when compering amino acid sequences
		- Dehydrogenase: H + OH = H20
	- PAM #: # number of mutations per 100 amino acid, e.g. 250 mean 250 mutations per 100, or 2.5 per amino acid (its not on DNA level, but amino acid level) so an amino acid could have changed multiple times L->K->R for instance
	- Blosum:# # percent identity of proteins used, e.g. Blosum 62 (most used, standard in BLAST)
	- Scoring matrix: Higher score for match mean there are less options of similar amino acid to be substituted in
	- Rule in DB search: extend the as long the score is above 0

	- Raw score (when aligning): match/mismatch score + gap penalty (opening and per gap)
	- y-axis: numb. of alignments, x-axis: (raw)score, percentage of alignments with a given score or better, the probability of the alignment being significant
	- bit score (normalize the score): s' = [lambda*S - ln(k)]/ln(2); lambda and k depend on scoring matrix and gap penalties used, make it possible to compare scores between scoring-systems like PAM and Blosum
	- normalize score based on length and query(n) and length of the database(m): E = k*m*n*e^(-lambda*S), expectation value
		- length of query and database effect the expectation, the longer database the more likely to find similarities by chance
		- E values: <1e-100 indicates identical proteins, 
				<1e-5,1e-50> homologues proteins,
				>1e-1 maybe random.
	
	- DNA evolves a lot faster than proteins, so different DNA sequences can be the same protein sequence
	- tblastx: DNA query and DNB database, but in protein space

	- Position Specific Weight Matrix(PSWM, or Position weight matrix PWM): for a selection of multiple different sequences of some given length, find the relative occurrence of amino acids (letters) for each position (maximum 1), then for a query sequence (same length) constructed from one of the possible letters at each position and ADD the corresponding relative value before dividing by the sequence length. For many sequences you could find an average value, and then compare that to the value of a given sequence

Week 10 Clustering:
- Clustering: Groupe objects that are similar (in some sense)
	- A method of classification (unsupervised ML)

- A (good) clustering principle: The distance between a pair of members in a group should be smaller then the distance between members of different groups

- Example of data than can be clustered:
	- Gene expression levels in various samples (multi-dimensional values)
	- Sequences (alignment distance)

- Common approaches: Centroid based and Hierarchical

	- Centroid based:
		- Define centroids (mass centres)
		- Calculate distance to centroid
		- Use various definitions of distance
		- k-means: 
			- k numbers of clusters
			- Lloyd's K-means algorithm:
				1. Initially choose k random datapoints as centres
				2. Assign each point to its nearest centre
				3. Calculate the centroid (mass centre) of each cluster
				4. Assign the points to its nearest centre of the new centres
				5. Repeat 3. and 4. until some threshold is met, e.g. distance of centroid displacement
		
		- Soft or fussy clustering: Each cluster is assign a colour, for instance red and blue, and the points given a value of each as a softmax vector, e.g. point1 (red: 0.80, blue: 0.20), point2 (0.98, 0.02), instead of a hard assignment

	- Hierarchical:
		- Uses distance between points
		- Types:
			- Agglomerative: 
				1. Every point start as their own cluster
				2. Then merges with the nearest cluster (point at this point) to create new cluster
				3. Continue until some specified number of clusters are met, or all points are within same cluster

			- Divisive: 
				1. All points is within the same cluster root/supercluster
				2. Then for each point calculate the distance to every other point and find average distance
				3. The point with the highest average distance become the starting point of a new cluster
				4. Then for each point check average distance to members in the old cluster vs the new
				5. Based on average distance, assign points to cluster
				6. For next iteration split the cluster with the furthest away member
				7. Meaning if we want 10 clusters we need to do 9 iterations of the algorithm

- Linkage criteria: How to compare distance between to clusters A and B:
	- Single linkage: Smallest distance between any point in A to any point in B
	- Average linkage: Between any point in A to any point in B (UPGMA)
	- Maximum linkage: Largest distance between any point in A to any point in B

	- Keep distance matrix updated as clusters are formed
	- UPGMA also used in phylogenetic tree reconstruction

- Distance metrics:
	- Euclidean
	- Manhattan
	- Maximum distance

	For sequences:
		- Total number of differences (mutations + indels)
		- Fraction (or percentage) of different residues

- Sequence clustering:

		- UCLUST algorithm:
			- Clusters defined minimum sequence similarity (e.g. 97%) with centroid representative sequence

			2. Choose first sequence as centroid for the first cluster
			3. Process the remaining sequences in order:
				3.1. Sequences with given similarity to any centroid sequence become member
				3.2. If no centroid similar enough, the sequence forms the centroid of a new cluster
